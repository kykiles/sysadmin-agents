## Навык: администрирование хоста

Ты выполняешь команды на самом сервере. Есть два инструмента:
- `host_query` — **только чтение** (iptables -L/-S, df, du, ss, ip show,
  systemctl status, journalctl, free, uptime). Безопасен, выполняется сразу.
- `shell_exec` — **любые команды, включая изменяющие**. Опасен, система запросит
  подтверждение пользователя автоматически.

### Общие правила
- Передавай команду списком аргументов: ["iptables", "-L", "-n"], а не строкой.
- Для диагностики/просмотра всегда используй `host_query` — без подтверждения.
- Если `host_query` вернул ошибку «команда не входит в список read-only» —
  значит операция изменяющая: используй `shell_exec`.
- Изменения (добавить правило, restart сервиса и т.п.) — только через `shell_exec`.
- Не выдумывай вывод — сообщай только фактический stdout/stderr и код возврата.

### Плейбук: firewall (iptables)
Порядок безопасной работы с правилами:
1. Осмотр текущих правил: `iptables -L -n -v` и `iptables -S`.
2. Перед изменением зафиксируй, что именно правило делает и не отрежет ли доступ
   (особенно правила на INPUT/22 порт SSH).
3. Добавление: `iptables -A INPUT -p tcp --dport 80 -j ACCEPT`.
4. Удаление по номеру: сначала `iptables -L INPUT --line-numbers`, затем
   `iptables -D INPUT <N>`.
5. Никогда не ставь политику DROP на INPUT, не убедившись, что SSH разрешён явным
   правилом выше по цепочке — иначе потеряешь доступ к серверу.

### Плейбук: systemd
- Статус: `systemctl status <unit>`; логи: `journalctl -u <unit> -n 100 --no-pager`.
- Управление (`start`/`stop`/`restart`) — только по явной задаче пользователя.

### Плейбук: диски
- Использование: `df -h`; крупные каталоги: `du -sh /path/*`.
