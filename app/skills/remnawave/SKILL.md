## Навык: панель remnawave

Ты работаешь с панелью VPN remnawave через её API. Два инструмента:
- `rw_query(script, args)` — **чтение**, выполняется сразу.
- `rw_action(script, args)` — **мутации** (меняют доступ платящего клиента), система
  запрашивает подтверждение автоматически.

Аргументы всегда передавай списком: `rw_query("user-find", ["alice"])`.

### Каталог скриптов

**Чтение (`rw_query`):**
- `user-find <query>` — поиск по username, telegram_id или uuid → краткая карточка.
- `user-get <uuid>` — полная карточка (статус, срок, трафик/лимит, устройства, sub-ссылка).
- `user-traffic <uuid>` — использовано/лимит/стратегия сброса.
- `user-devices <uuid>` — список HWID-устройств.
- `nodes` — ноды: онлайн/офлайн, статус, юзеры онлайн.

**Мутации (`rw_action`, каждая — с подтверждением):**
- `user-extend <uuid> <days>` — продлить подписку на N дней (от max(текущий срок, сейчас)).
- `user-enable <uuid>` / `user-disable <uuid>` — включить / приостановить доступ.
- `user-revoke <uuid>` — отозвать подписку (ротация ключей и sub-ссылки; старая перестаёт работать).
- `user-reset-traffic <uuid>` — обнулить счётчик трафика.
- `hwid-reset <uuid> [hwid]` — сбросить конкретное устройство или все.

### Правила
- **Всегда сначала найди пользователя и покажи текущее состояние** (`user-find` →
  `user-get <uuid>`), и только потом мутация. Это план: пользователь видит, что и кому меняем.
- Мутацию вызывай по `uuid`, а не по имени — избегаешь ошибки с однофамильцами.
- Не выдумывай результат — возвращай фактический JSON от API и краткий вывод на русском.
- Создание новой подписки/пользователя в этот навык НЕ входит — это делает бот продаж.

### Плейбук: «клиент оплатил, а доступа нет»
1. `user-find <telegram_id или username>` → возьми `uuid`, посмотри `status`/`expireAt`.
2. Если `status=DISABLED` или срок истёк — покажи это; при необходимости `user-enable`
   и/или `user-extend`.
3. Если жалуется на «не подключается» — `user-devices` (лимит устройств) и `nodes` (ноды живы?).

### Плейбук: «сменить ключ / скомпрометирован»
- `user-get <uuid>` → `user-revoke <uuid>`: выдаст новую sub-ссылку, старую сообщи как недействительную.
