## Навык: память об инфраструктуре

У тебя есть общая память команды о фактах окружения. Три инструмента:
- `recall_facts` — вспомнить факты (все, по области `scope` или по подстроке `query`). Безопасно.
- `remember_fact` — записать устойчивый факт (`scope`, `key`, `value`). Перезаписывает факт с тем же `scope`+`key`. Безопасно.
- `forget_fact` — удалить один факт по `scope`+`key`. Безопасно.
- `forget_facts` — удалить ВСЕ факты области `scope` (напр. хост выведен из эксплуатации). Безопасно.

### Когда вспоминать
- Перед действием на конкретном хосте — сначала `recall_facts(scope=<хост>)`, чтобы не переоткрывать уже известное (версии, пути, порты, топология).

### Что записывать
Записывай только **устойчивые** факты, а не эфемерный вывод команд:
- версии сервисов, пути к конфигам, номера портов, расположение данных, топология.
- НЕ записывай: текущую загрузку CPU, список процессов, разовый вывод логов.

Помечай вид знания параметром `kind`:
- `stable` (по умолчанию) — то, что меняется редко и осмысленно: топология, пути к конфигам,
  принятые решения («на этом хосте iptables, а не ufw»).
- `snapshot` — то, что дрейфует само: версии сервисов, номера портов, размеры дисков.
  Такие факты перепроверяются раньше, поэтому не помечай их `stable`.

### Как записывать
- `scope`: `"global"` для фактов обо всей системе, либо идентификатор хоста (адрес/имя), если факт про конкретный сервер.
- `key`: короткий snake_case (`postgres_version`, `nginx_conf_path`, `ssh_port`).
- `value`: само значение, кратко.
